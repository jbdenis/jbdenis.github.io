<h2>handling files</h2>
<p>
For many years I used some small Perl scripts to tidy my files up. I
finally gathered all these functionalities within a unique general
script named <a href="gefi">gefi</a> which can be called through more targetted scripts, namely:
<ul>
<li><a href="jeter">jeter</a> to throw in the thrash a set of files.</li>
<li><a href="sauver">sauver</a> to make precautionary savings of files.</li>
<li><a href="padb">padb</a> to replace spaces :+( in files names.</li>
<li><a href="nettoyer">nettoyer</a> to remove useless files.</li>
<li><a href="norfo">norfo</a> to normalize the names of my picture files.</li>
</ul>
</p>
<p>
Inline help of all these scripts is obtained calling them without arguments.
</p>

<h2>fitdistrplus and mc2d</h2>

<p>Distributions, Tools for Data Fitting to Univariate Distributions and Tools
for 1D and 2D Monte-Carlo Simulations.</p>

<p>An <strong>R</strong> package project for fitting distributions, assessing
variability and uncertainty in the context of food safety modelling. The
downloads are available at <a
href="http://r-forge.r-project.org/projects/riskassessment/">R-Forge</a>. These
tools were imagined and written by R&eacute;gis Pouillot and Marie-Laure
Delignette.</p>
<a name="rebastaba"></a> 

<h2>comment and paquet</h2>

<p>Standardizing general comments in R functions and extracting them to produce
manual files : description of the <a href="comment.r">format</a>, Perl <a
href="comment.pl">script</a>. The improvements of this last version are due to
R&eacute;gis Pouillot.</p>

<p>This script is used in another script to automatically produce R packages.
The name is <a href="paquet.pl">paquet.pl</a>, some details are given in its
comments. It was built for the generation of RebaStaBa.</p>

<p>If you want to try it with a complete example, just save and expand <a
href="pr.zip">pr.zip</a>. A named <strong>paquet</strong> directory will be
created including <em>comment.pl</em> and <em>paquet.pl</em> and the necessary
files into a <em>perso</em> directory for the package <strong>rs00</strong> of
the miscellaneous utility functions of rebastaba (you can find some of them of
use too). To check the package and build its documentation pdf file, just type
from the <em>paquet</em> directory <code>paquet.pl rs00 cd</code>. </p>


<h2>Bayesian networks</h2>

<p><big>rebastaba</big> for the manipulation of Bayesian networks
within R is now an abandoned project. Its inheritance is a set of

named <a href="rbsb_0.3-6.tar.gz">rbsb</a>.
</p>
<p>
A further issue is a dedicated set of functions to handle Gaussian
bayesian networks under the name of <a href="rbmn_0.4-6.tar.gz">rbmn</a>.
</p>

<h2>extracting from MCMC computations</h2>

<p>When using <a href="http://www-fis.iarc.fr/~martyn/software/jags/">Jags</a>
you can produce outputs too large for reading them in R. As it is not that
simple to know in advance which size to give to the burnin phase and which
thinning frequency to use in advance, one can be obliged to run several times
the same MCMC. Using <a href="jaxtrai.pl">jaxtrai.pl</a> allows you to handle
much larger outputs. </p>

<h2>intera and biareg</h2>

<p>Modelling interaction between two factors was a main focus of my research
activities many years ago when I was collaborating with plant geneticists
interpreting genotype by environment interactions. INTERA, written with the
late Guy Decoux, is a set of Pascal programs running on DOS. To use them you
need to know how to edit a text file (kind of script files) and how to launch
from a terminal this kind of programs. You can get the <a
href="interaD12.zip">binaries</a> of the programs or the <a
href="interaD12345.zip">binaries and soures and documentations</a> in zipped
files. Unfortunately, except an extended summary, the main part of the <a
href="decoux.1991.intera.notice.zip">manual</a> is written in French.</p>

<p>More recent development was undertaken in SPlus with the BiaReg project. <a
href="biareg.zip">Source files</a> are available, also is a companion <a
href="denis.1998.biadditive_regression.two_way_interaction.pdf">manual</a>. The
flaw is that BiaReg does not deal with missing values. It is not so much work
to encapsulate BiaReg with an EM or ALS algorithm, Vincent Foucteau did it
during his PhD but this was never published.</p>

<p>I cannot give any help about the use of intera or biareg.</p>
